Broadcast 值获取流程

以步骤 2 处为例。

在 value ()方法内部其实调用的是 getValue ()方法，而 getValue ()方法在 Broadcast 类内部是一个抽象方法，所以其实根本上调用的是 TorrentBroadcast 类内部的 getValue()方法。

在 getValue ()方法内部首先会查看_value 中是否缓存有（_value 是个软连接），如果有就直接返回即可。没有则需要通过 readBroadcastBlock ()方法进行读取相应的 block。

在 readBroadcastBlock ()内部首先会尝试从 broadcastManager 的 cachedValues 中获取（cachedValues 本质上是一个 ReferenceMap 里面的 key 是一个强引用，value 是一个弱引用），如果有就会直接返回，否则就会执行 getOrElse 的 default 方法。

在 default 方法内首先会尝试使用 getLocalValues 方法从 BlockManager 的本地进行获取，如果可以从本地获取就会将其存入 cachedValues 并释放 BlockManager 的锁，然后返回。

getLocalValues 方法其实就是根据存储的级别分别从 memoryStore 或者 diskStore 中进行获取。

如果本地获取不到就需要尝试使用 readBlocks 方法，将 chunk 数据读取到并将数据转换为 broadcast 的 value 对象，并将该对象放入 cachedValues 中。
