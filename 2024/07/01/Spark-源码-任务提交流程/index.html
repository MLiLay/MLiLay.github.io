Spark-源码-任务提交流程（Yarn）

发表于2024-07-01|更新于2024-07-25|Spark
|阅读量:

以下 spark 版本为 3.3.2

Spark 源码

主要为 Spark 任务的启动

Spark 运行流程 (Yarn)

当用户调用 SparkSubmit 脚本提交任务时，会调用 org. apache. spark. deploy. SparkSubmit. Scala 进行程序的提交。

SparkSubmit. Scala

SparkSubmit 被调用后首先会调用 Object 半生类中的 main 方法进行启动，mian 方法内主要就是创建一个 submit 对象，然后调用 doSubmit()方法进行启动。

doSubmit（）

主要用途：对命令行输入的参数进行匹配和初始化，存储到一个 Map 中，最后根据得到参数中的 action 执行不同的操作：submit，kill，requestStatus，printVersion.

在 submit 方法中是调用 runMain ()方法，开启任务的运行。

runMain ()

runMain ()方法中包含几个关键步骤。

  • prepareSubmitEnvironment ()方法准备 Spark 运行的环境，主要是根据用户的输入参数与 Spark 需要的参数的一个 match 的过程，最后返回 childArgs, childClasspath, sparkConf, childMainClass 四个参数，用于后续使用。

  • 创建获取到的 childMainClass 创建在本地创建一个 Application 作为程序的整个提交任务的入口，childMainClass 中是通过用户在输入 master 参数的值在 prepareSubmitEnvironment 中通过 match 得到的之前定义在 SparkSubmit. scala 中的各个 SparkApplication 的实现类。

  • 创建的 Client（app）调用 start ()方法，start ()方法内部 new Client 实例，然后调用其 run ()方法。之后在 run ()方法内部调用 submitApplication ()方法提交应用。

  • 在 submitApplication ()方法内部主要分为几步。

  • 验证资源的正确性。

  • 连接 RM，创建一个新的 Client 与 RM 进行通讯

  • 创建 Client 用于提交到集群的命令，这其实就是最本质的任务，其中包含了创建 ApplicationMaster 等。

  • 最后 yarnClient 提价客户端命令（任务）

ApplicationMaster. scala

在 yarn 模式下，yarn 进行任务的调度，在空闲的 executer 领取到资源，创建 ApplicationMaster。ApplicationMaster. scala 内描述的就是 ApplicationMaster 做的事情。

  • 在 mian 主函数内首先会创建一个 ApplicationMaster

  • 验证用户权限

  • 最后调用创建的 ApplicationMaster 实例中的 run ()方法。

  • 在 run ()方法中会创建一个关闭时用的钩子，用于在 SparkContext 完全关闭后清理 staging 并从 RescourceManager 中注销应用

  • 之后如果是集群模式会运行 runDriver ()方法

  • 在 runDriver ()方法中首先会使用 startUserApplication ()创建一个一个用户线程，这个线程的名字叫 Driver，用于整个程序的驱动。

  • 在等待环境信息配置完成后，会在 ApplicationMaster 中创建一个 Client 负责与 ResourceManager 进行通信，注：client 是一个成员变量早就创建过了，这里只是在 RescourceManager 中进行注册

  • 使用 createAllocator ()方法申请资源，并对申请到的 Container 进行过滤选取之后运行对应的任务。

    • 首先获取运行所需要的资源数。

    • 然后创建一个 allocator 获取资源并执行任务。

  • 在 allocateResources ()方法内部首先会通过 amClient 与 RM 进行通信，获取到全部可用的 Container

  • 使用 handleAllocatedContainers ()方法对得到的 resource 进行处理。本质上是一个 match 的过程，最后 match 成功的 executer 会被存储在 allocatedContainers: Seq[Container]中。

  • 最后在 handleAllocatedContainers ()方法中会调用 runAllocatedContainers ()方法对符合要求的 Container 进行资源的申请和 Executor 的创建。

可以看到在 Container 的启动中其实是将其放在了一个线程池中由线程负责其启动。

接下来我们看 run 方法

内部其实就是在 NM 上启动Container

在 startContainer 内部其实启动org.apache.spark.executor.CoarseGrainedExecutorBackend

在创建 org. apache. spark. executor. CoarseGrainedExecutorBackend 的过程中会调用其 main 方法，在 main 方法内部会调用其 run 方法，在 run 方法内部其实就是创建 Executor 及其所需的 env（其中包含 blockmanager 等一些组件）。
